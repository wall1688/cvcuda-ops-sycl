# CV-CUDA CvtColor 算子 SYCL 移植总结

> 本文件说明:做了什么、文件夹里是什么、怎么编译运行、正确性与性能如何。
> 环境(Intel 开发板、icpx / OpenCL 后端、两段 SSH 工作流等)已在体素化算子任务中打通,本任务直接复用,不再赘述。

## 一、CvtColor 算子移植

### 1. 代码理解
1. 阅读了 CV-CUDA 0.16.0 的 `src/cvcuda/priv/legacy/cvt_color.cu`(2010 行)及依赖(`OpCvtColor.{hpp,h}`、`cuda_tools/{TensorWrap,SaturateCast,TypeTraits}.hpp`、测试 `CvtColorUtils.cpp` 的 gold 族参考)。
2. **"全量"的真实含义**:CV-CUDA 的 `funcs[]` dispatch 表里,XYZ/Lab/Luv/HLS/Bayer/BGR565/555/mRGBA 全部是 `0`(返回 INVALID_PARAMETER,**无 kernel**)。原版仅实现 6 个码族。忠实移植 = 移植这 6 族 + 对未实现码返回同样 unsupported(不凭空发明)。
3. 6 个码族:
   - A 通道互换/alpha(码 0-5):纯通道重排,codes 0,1 复制±alpha,2,3,4,5 反转 [2,1,0]±alpha。
   - B 灰度(6-11):BGR→GRAY(定点 Q15 CV_DESCALE + float)、GRAY→BGR(复制)。
   - C 打包 YUV(82-85):Q14 定点 + float 正/逆变换。
   - D HSV(40,41,54,55,66,67,70,71):uchar 定点 hdiv/sdiv 表 + 位运算 hue;float 6 段 sector LUT(3 个 int32 位掩码)。
   - E YUV420(90-106,127-134,140-147):NV12/NV21 半平面 + I420/YV12 平面,4:2:0 子采样。
   - F YUV422(107-124,不含 109/110/113/114):YUY2/YVYU/UYVY 4:2:2,2 像素共享 U/V。
4. **布局**:严格 NHWC。关键推导 `Tensor2DWrap::operator[](int2 c)` → `doGetPtr(c.y, c.x)`,行主序连续。YUV420/422 是标量 [N,1.5H,W,1] / [N,H,2W,1]。用原生 USM + stride 替代 nvcv TensorWrap。
5. **dtype 矩阵(匹配原版)**:A + GRAY→BGR = 全 dtype;BGR→GRAY + 打包 YUV = 8U/16U/32F;HSV = 8U/32F;YUV420/422 = 8U only。16F→uint16_t 整数路径;64F→double。
6. CPU gold(`CvtColorUtils.cpp`)覆盖全部 6 族,作正确性参照。

### 2. CUDA → SYCL 核心映射
| CUDA (CV-CUDA) | SYCL (本移植) |
|---|---|
| `__global__` + `<<<grid,block>>>`(per-pixel,decode n/y/x) | `q.parallel_for(range<1>{total}, λ)`,内部分解 `n/y/x` |
| `Tensor3DWrap/4DWrap` `.ptr(b,y,x[,c])` | 原生 USM 指针 + `((n*H+y)*W+x)*C+c` 索引 |
| `cuda::max/min` | `sycl::max/min`(显式转 int 保 OpenCL 可移植) |
| `cuda::SaturateCast<T>` | `sat_int`(int→无符号 clamp)/ `sat_cast_f`(float→整,`sycl::rint` 半偶舍入 + clamp,匹配 PTX `cvt.rni`) |
| `CV_DESCALE`、ITUR_BT_601 定点系数、`Alpha<T>` | 逐行搬至 `cvt_helpers.hpp` |
| `if constexpr is_integral` 定点/float 双路径 | 保留;dtype host 端模板派发 |
| `cudaMemcpy2DAsync`(YUV2GRAY_420) | `q.memcpy` 逐 sample 拷 Y 平面 |
| `cudaStream_t stream` | `sycl::queue*`(nullptr 用内部 queue,in-order + `VOX_PROFILE` 开 `enable_profiling`) |
| kernel 返回 void | `parallel_for` 返回 `sycl::event`,保存供 `last_profile()` 计时 |
| 原子 / 共享内存 / 纹理 | **无** —— 纯 per-pixel 寄存器运算 |

> 本仓版本为自包含 header-only:`nv::linear_launch` → 原生 `q.parallel_for`,入参校验直接 `return false`(原项目 `Assertf/Asserts` 宏未使用,故不依赖外部 `common/check.hpp`)。kernel 数学逐字不变。

### 3. 写出文件
```
OpColorCvt/
├── README.md                    # 工作说明 + 使用指南
├── 总结.md                      # 本文件(移植总结)
├── build.sh                     # 编译脚本(编 2 个 test 二进制;kernel header-only)

├── cvt_helpers.hpp              # SaturateCast/Alpha/TypeMax/TypeMin/NHWC ptr/颜色常量/cv_descale
└── test/
    ├── cvt_gold.hpp             # CPU gold(移植自 CvtColorUtils,6 族)
    ├── test_cvt.cpp             # 71 码逐族逐 dtype vs gold + 负向测试

```
**组织原则**(同 OpResize / OpNMS):kernel 全部 header-only(`colorcvt.hpp`),test 直接 include,`build.sh` 只编 2 个 test 二进制;CPU 参考逻辑移植自 CV-CUDA 自带的 `CvtColorUtils`。原始 NVIDIA 源在 CV-CUDA 0.16.0 仓库,此处不重复收录。

### 4. 编译调试
1. kernel header-only(`colorcvt.hpp`),test 直接 include,`build.sh` 只编 2 个 test 二进制,无需单独编 kernel。
2. 编译选项 `icpx -fsycl -std=c++17 -O2 -g -I. -ffp-model=precise -fno-fast-math -fimf-precision=high`(由 `build.sh` 编 2 个目标)。
3. 为 OpenCL 后端可移植性,`sycl::max/min` 显式转 int。
4. Stage 0 修了一个错:测试里 `shared_ptr` 误用 `.` 调 `forward`(应 `->`)。分 6 阶段(Stage 0-5)逐族移植+验证,每阶段板上 build/run 通过再进下一阶段,无新坑。

### 5. 正确性验证
71 个码逐族逐 dtype vs CPU gold,全部 `=== ALL TESTS PASSED ===`:
- A 族:U8/U16/S32/F32/F64 逐 bit 一致(纯重排,tol 0)。
- B 族:BGR→GRAY U8(tol 1)/U16(tol 2)/F32(ERR1_4);GRAY→BGR exact。定点 CV_DESCALE 与 gold float+truncate 在容差内。
- C 族:U8(tol 1)/U16(tol 2/1)/F32(ERR1_4)。SaturateCast 验证。
- D 族 HSV:U8(tol 1)/F32(BGR2HSV ERR2_3 circular hue / HSV2BGR ERR1_4)。uchar 定点 hue + float sector LUT 验证。
- E 族 YUV420:YUV→BGR(tol 2)/BGR→YUV(tol 1)/GRAY(exact)。semi-planar + planar 4:2:0 子采样布局逐位对齐。
- F 族 YUV422:YUV→BGR(tol 2)/GRAY(exact)。4:2:2 宏像素路由验证。
- 负向:16F 加 alpha 拒绝、未实现码(50)拒绝、通道数不符拒绝。

## 二、性能

### 1. profile 方法
SYCL event profiling(`VOX_PROFILE=1` opt-in,`last_profile(ns)` 提取 kernel device 时间)+ steady_clock wall。`test_cvt_profile.cpp`:1280×720、N=1、20 runs 取 best。

### 2. profile 数据(best kernel ms / wall ms / Mpix/s)
```
BGR2BGRA   (A)   0.097 / 0.137 /  9513
BGR2RGB    (A)   0.196 / 0.230 /  4709
BGR2GRAY   (B)   0.079 / 0.096 / 11703
BGR2YUV    (C)   0.174 / 0.208 /  5288
BGR2HSV    (D U8) 0.128 / 0.161 /  7199
BGR2HSV    (D F32)0.161 / 0.245 /  5715
YUV2BGR_NV12(E)  0.155 / 0.186 /  5946
YUV2GRAY_420(E)  0.017 / 0.023 / 55126   ← memcpy Y 平面,带宽极限
YUV2BGR_YUY2(F)  0.150 / 0.183 /  6131
```

### 3. 性能极限分析
- **内存带宽受限**:每像素读一次写一次,NHWC 连续访问已合并(coalesced)。720p 计算核 4.7-11.7 Gpix/s,memcpy 55 Gpix/s。
- overhead(wall−kernel)仅 0.03-0.08ms(launch + wait),单 kernel 无中段 host sync。
- 原版 `color_conversion_common` 的 tile+RPT 驱动主要利于**计算受限** kernel 的寄存器复用;但本算子**带宽受限 + 访问已合并** → tile+RPT 收益可忽略,不做。
- 结论:per-pixel launch 已接近该实现带宽极限。进一步提速需算法级改动(融合上下游算子、减少中间写回),属另一阶段工作(同 voxelization/NMS)。

### 4. CPU↔GPU speedup 对比

为回答"算子移植有没有做 CPU/GPU 对比验证",给 `test_cvt_profile.cpp` 加了 env 门控 `VOX_CPUTIME=1`(默认关)的 CPU gold 参考计时。CPU 参考为朴素单线程实现(与正确性测试同一份 `cvt_gold.hpp` 的 `cvt_gold_dispatch<T>`,6 族派发),`speedup = cpu_ms / gpu_best_kernel_ms`(SYCL event profiling, best-of-20;CPU 1 run)。profile 把同一 `host_in` 字节喂给 gold(F32 case reinterpret 为 float),gold 与 GPU 做相同的逐像素工作。

运行:

```
ONEAPI_DEVICE_SELECTOR=opencl:gpu VOX_CPUTIME=1 ./test/test_cvt_profile
```

实测(Intel Arc [0xb080],1280×720 N=1,14 码,2026-07-02):

| BGR2BGRA   | A | 0.0828ms | 1.950ms | 23.6× |
| BGR2RGB    | A | 0.1524ms | 1.365ms | 9.0× |
| BGR2GRAY   | B | 0.0696ms | 1.378ms | 19.8× |
| GRAY2BGR   | B | 0.0703ms | 1.735ms | 24.7× |
| BGR2YUV    | C | 0.1329ms | 4.709ms | 35.4× |
| YUV2BGR    | C | 0.1784ms | 4.305ms | 24.1× |
| BGR2HSV (U8)  | D | 0.1358ms | 10.349ms | 76.2× |
| BGR2HSV (F32) | D | 0.1560ms | 15.223ms | 97.6× |
| HSV2BGR    | D | 0.1807ms | 5.212ms | 28.8× |
| YUV2BGR_NV12  | E | 0.1298ms | 5.111ms | 39.4× |

说明:
- speedup 反映"GPU vs 朴素单线程 CPU"量级,**非优化基线**(GPU 自身性能/瓶颈见 4.2/4.3)。
- HSV(D 族)CPU gold 定点 hue + sector LUT 逐像素最慢,speedup 最高(76×–98×);纯通道重排(A 族)CPU gold 只搬数据,speedup 低(9×–24×)。
- YUV2GRAY_420 / YUV2GRAY_YUY2 是 memcpy Y 平面路径,GPU 已达带宽极限(~55 Gpix/s),CPU gold 也只搬小数据量,speedup 最低(4×–5×)。
- CPU 参考非 NVIDIA bit-diff(gold 源自 CV-CUDA `CvtColorUtils` 独立参考,见三章),逻辑逐字照抄。

## 三、验证局限(诚实声明)
1. **自洽验证,非对齐验证**:正确性测试用的是我自己移植的 CPU gold(源自 CV-CUDA `CvtColorUtils`,与 `.cu` kernel 是两份独立实现),证明「SYCL kernel == 我的 CPU 参考」,**不直接证明 == NVIDIA CV-CUDA 实际输出**。若 kernel 与 gold 都从同一份源理解错,二者会一致地错,测试仍 PASS。缓解:kernel 与原 `.cu` 逐行对照;gold 取自源仓独立参考。
2. **未与 NVIDIA CV-CUDA 做 bit 级 diff**:真正的金标准是同参数喂 NVIDIA `cvcudaCvtColor` 和本 SYCL kernel 比输出。这一步没做(需 NVIDIA GPU + 编译 CV-CUDA,Intel Arc 跑不了 CUDA 版)。
3. **容差覆盖 FP 差异**:HSV/YUV float 路径含除法,Intel Arc GPU float 除法非 IEEE-754 正确舍入(倒数近似);这里用 CV-CUDA 原版容差覆盖,未单独做边界归类(不像 NMS 那样做 IoU 边界分析)。归因是推断,非单独证明的根因。

## 四、与其它算子移植的差异小结
| 维度 | 体素化 | NMS | CvtColor |
|---|---|---|---|
| kernel 数 | 3 | 1 | 11 个模板(6 族) |
| 原子/hash 表 | 有 | 无 | 无 |
| 优化阶梯 | opt1-4,3.5-3.8× | 无阶梯 | 无阶梯(带宽受限) |
| 规模/复杂度 | 3 kernel | 单 kernel ~10 行 | 6 族 / 71 码,HSV 位运算 hue + YUV420 双布局子采样最难 |
| overhead | ~0.10-0.12ms | ~0.01ms | 0.03-0.08ms |
| 瓶颈 | 随机 hash 访存,延迟受限 | O(N²) 随机 box 读 | 带宽受限,访问已合并 |
| FP 边界 | 点除法→体素 ±1 平移 | IoU 除法→阈值翻转 | HSV/YUV float 除法,容差覆盖 |

共性:都用 OpenCL 后端;都复用 `make_internal_queue` + `VOX_PROFILE` event 计时;FP 差异都归因 Intel GPU 除法非 IEEE 舍入(设备固有,非 bug);性能极限都属「需算法级改动才能突破」。
