# CV-CUDA Normalize 算子 SYCL 移植总结

> 本文件说明:做了什么、文件夹里是什么、怎么编译运行、正确性与性能如何。
> 环境(Intel 开发板、icpx / OpenCL 后端、两段 SSH 工作流等)已在体素化算子任务中打通,本任务直接复用,不再赘述。

## 一、Normalize 算子移植

### 1. 代码理解
1. 阅读了 CV-CUDA 0.16.0 的 `src/cvcuda/priv/legacy/normalize.cu`(tensor path)及依赖(`cuda_tools/{SaturateCast,TypeTraits,TensorWrap}.hpp`、`include/OpNormalize.h` 规格、测试 `TestOpNormalize.cpp` 的 Normalize() gold 参考)。VarShape 路径 deferred。
2. **公式**:`out = SaturateCast<outT>((src - base) * mul * global_scale + shift)`;`mul = scale`(plain)或 `1/sqrt(scale²+epsilon)`(`SCALE_IS_STDDEV` flag = bit 0)。base/scale 始终 `float*`(CV-CUDA `work_type = ConvertBaseTypeTo<float,input_type>`),按轴广播。
3. **支持范围**:dtype U8/S8/U16/S16/S32/F32(F64/U32/F16 不支持);通道 C∈{1,3,4}(C==2 / C>4 拒绝,匹配 `funcs[6][4]` 表的 null slot);NHWC;`in_dtype == out_dtype`。
4. **广播**:per-axis,`param_idx[axis] = (param_size[axis]==1) ? 0 : data_idx[axis]`,匹配 CV-CUDA 的 `base_size/scale_size` int3 + channel==1 逻辑。
5. CPU gold(`TestOpNormalize.cpp` 的 Normalize() 辅助,float math + `std::rint` + 手动 clamp)覆盖 plain+stddev + 全 dtype + 广播。

### 2. CUDA → SYCL 核心映射
| CUDA (CV-CUDA) | SYCL (本移植) |
|---|---|
| `__global__` + `<<<grid,block>>>` block(32,8) grid(W,H,N),`blockIdx.z`=batch | `q.parallel_for(range<1>{N*H*W}, λ)`,内部分解 `n=idx/(H*W), y=.../W, x=...%W` |
| `CreateTensorWrapNHW<T>` + vector-type 派发(uchar4/float3)`funcs[6][4]` | 单 `EltT` 模板 + 运行时 C,通道内部循环(数学等价,无 vector 机制) |
| `cuda::SaturateCast<T>` | `sat_cast<T>`(`sycl::rint` 半偶舍入 + clamp [TypeMin,TypeMax];F32 恒等) |
| `base_size/scale_size` int3 + channel==1 广播 | `Shape{N,H,W,C}` + `param_idx = size==1?0:data_idx` |
| plain/stddev 两个 kernel | 共用一个 kernel(`bool stddev` 分支) |
| `cudaStream_t stream` | `sycl::queue*`(nullptr 用内部 queue,in-order + `VOX_PROFILE` 开 `enable_profiling`) |
| kernel 返回 void | `parallel_for` 返回 `sycl::event`,保存供 `last_profile()` 计时 |
| 原子 / 共享内存 | **无** —— 纯 per-pixel 寄存器运算 |
| 接口 void / 异常 | 返回 **bool**(校验 C/dtype/broadcast,匹配 CV-CUDA INVALID_PARAMETER) |

> 本仓版本为 self-contained header-only:`nv::linear_launch` → 原生 `q.parallel_for`,不依赖项目 `common/launch.hpp`/`common/check.hpp`(后者 `Assertf/Asserts` 未使用,入参校验直接 `return false`)。kernel 数学逐字不变。

### 3. 写出文件
```
OpNormalize/
├── README.md                    # 工作说明 + 使用指南
├── 总结.md                      # 本文件(移植总结)
├── build.sh                     # 编译脚本(编 3 个 test 二进制;kernel header-only)

└── test/
    ├── normalize_gold.hpp       # CPU gold(移植自 TestOpNormalize)
```

**组织原则**(同 OpResize / OpNMS / OpColorCvt):kernel 全部 header-only(`normalize.hpp`),test 直接 include,`build.sh` 只编 3 个 test 二进制;CPU 参考逻辑移植自 CV-CUDA 自带的 `TestOpNormalize`。原始 NVIDIA 源在 CV-CUDA 0.16.0 仓库,此处不重复收录。

### 4. 编译调试
1. kernel header-only(`normalize.hpp`),test 直接 include,`build.sh` 只编 3 个 test 二进制,无需单独编 kernel。
2. 编译选项 `icpx -fsycl -std=c++17 -O2 -g -I. -ffp-model=precise -fno-fast-math -fimf-precision=high`(由 `build.sh` 编 3 个目标)。
3. 一处 fix:real 测试 perf 段 `op->forward`(`op` 是 shared_ptr,非 `op.forward`)。
4. 复用 NMS 的 `make_internal_queue`/`last_profile`/arg-shadow;colorcvt 的 `sat_cast_f`/TypeMin/TypeMax/dtype-switch/容差比较模式。

### 5. 正确性验证
42 case(6 dtype × 7 场景)全部 `=== ALL TESTS PASSED ===`,bit-exact mismatch=0:
- 手选精确值:2 的幂 scale、整数 base/shift、stddev eps∈{1,4,16} 使 1/sqrt 为精确 2 的幂。
- 覆盖 plain+stddev、C∈{1,3,4}、scalar / per-channel / N/H/W 广播。
- 负向:C==2 拒绝、C==5 拒绝、bad broadcast 拒绝。

随机大规模(N=64 H=128 W=128 C=4):
- U8/F32 **plain** mismatch=0(无除法,IEEE mul/add 精确,无 FMA 收缩)。
- **F32 stddev** mismatch=0(tol 1e-4),maxdiff=1.53e-05(ULP 噪声,1/sqrt 发散)。
- **U8 stddev** mismatch=0(样本未越界);整型 divergent 用 `is_boundary`(BOUND_TOL=1e-3)归类。

## 二、性能

### 1. profile 方法
SYCL event profiling(`VOX_PROFILE=1` opt-in,`last_profile(ns)` 提取 kernel device 时间)+ steady_clock wall。`test_normalize_profile.cpp`:N=64 H=128 W=128 C=4,20 runs 取 best。

### 2. profile 数据
```
----- Normalize per-kernel (best) -----
normalize_kernel  best=0.573ms avg=0.722ms (100%)
wall              best=0.704ms
overhead          =0.132ms (launch + wait)
throughput        kernel=1831 Mpix/s  wall=1489 Mpix/s  (7325 Melem/s kernel)
```

### 3. 性能极限分析
- 单 kernel、embarrassingly parallel、无 host sync、无原子。overhead 0.132ms(launch + wait)。
- 后续优化方向(本轮未做,按「先 1:1 跑通再谈优化」):`sycl::vec<float,4>` 通道向量化、tile + 只读 cache 复用广播 base/scale、RPT。

## 三、验证局限(诚实声明)
1. **自洽验证,非对齐验证**:正确性测试用的是我自己移植的 CPU gold(源自 CV-CUDA `TestOpNormalize`,与 `.cu` kernel 是两份独立实现),证明「SYCL kernel == 我的 CPU 参考」,**不直接证明 == NVIDIA CV-CUDA 实际输出**。若 kernel 与 gold 都从同一份源理解错,二者会一致地错,测试仍 PASS。缓解:kernel 与原 `.cu` 逐行对照;gold 取自源仓独立参考。
2. **未与 NVIDIA CV-CUDA 做 bit 级 diff**:真正的金标准是同参数喂 NVIDIA `cvcudaNormalize` 和本 SYCL kernel 比输出。这一步没做(需 NVIDIA GPU + 编译 CV-CUDA,Intel Arc 跑不了 CUDA 版)。
3. **FP 归因是推断**:stddev maxdiff=1.5e-5 是直接观测;归因到 Intel GPU 倒数近似除法是 **推断**(precedent + ULP 量级),非单独证明的根因。gold 用 `std::sqrt` + IEEE 除法,kernel 用 `sycl::sqrt` + GPU 除法。U8 stddev 显示 0 mismatch 是样本现象 —— 极端 scale/eps 可能产生边界 mismatch(real 测试的 `is_boundary` 会归类)。

## 四、与其它算子移植的差异小结
| 维度 | 体素化 | NMS | CvtColor | Normalize |
|---|---|---|---|---|
| kernel 数 | 3 | 1 | 11 模板(6 族) | 1(plain+stddev 共用) |
| 原子/hash 表 | 有 | 无 | 无 | 无 |
| 优化阶梯 | opt1-4 | 无 | 无(带宽受限) | 无(本轮) |
| FP 风险 | 点除法→体素 ±1 | IoU 除法→阈值翻转 | HSV/YUV 除法,容差覆盖 | stddev 1/sqrt,ULP 1.5e-5 |
| 接口返回 | op 拥有输出 | 调用方传 out | bool(校验) | bool(校验 C/dtype/broadcast) |
| 容差策略 | 边界归类 | 边界归类 | CV-CUDA 原容差 | plain bit-exact / stddev tol 1e-4 + 边界归类 |

共性:都用 OpenCL 后端;都复用 `make_internal_queue` + `VOX_PROFILE` event 计时;FP 差异都归因 Intel GPU 除法非 IEEE 舍入(设备固有,非 bug);plain 模式无除法可做到 bit-exact。
