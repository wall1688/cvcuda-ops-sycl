# CV-CUDA Rotate 算子 SYCL 移植总结

> 本文件说明:做了什么、文件夹里是什么、怎么编译运行、正确性与性能如何。
> 环境(Intel 开发板、icpx / OpenCL 后端、两段 SSH 工作流等)已在体素化算子任务中打通,本任务直接复用,不再赘述。

## 一、Rotate 算子移植

### 1. 代码理解
1. 阅读了 CV-CUDA 0.16.0 的 `src/cvcuda/priv/legacy/rotate.cu`(tensor path)及依赖 `cvcuda/cuda_tools/InterpolationWrap.hpp`(NEAREST/LINEAR/CUBIC + BORDER_REPLICATE)。`rotate_var_shape.cu` deferred。
2. **算法**(backward inverse-map,per output pixel `(n,y,x)`):`dx=dstX-xShift, dy=dstY-yShift`(double);`srcX=(float)(dx*cos - dy*sin)`,`srcY=(float)(dx*sin + dy*cos)`;coeffs `[c,s,xShift,-s,c,yShift]`(`compute_warpAffine`)。在范围 `srcX>-0.5 && srcX<W && srcY>-0.5 && srcY<H` 内 → REPLICATE+interp 采样,否则 0。
3. **采样**:NEAREST `floor(coord+0.5)`;LINEAR 双线性 4-tap(`x1=floor(x),x2=x1+1`);CUBIC `GetCubicCoeffs`(a=-0.5)4×4-tap。`SaturateCast` = rint+clamp。
4. **支持范围**:dtype U8/U16/S16/F32;C∈{1,3,4}(C==2 null slot / C>4 拒绝);in/out H/W 可不同,batch N 同;NHWC。
5. **gold**:CV-CUDA rotate 不带独立 test gold,本 gold 自移植自 `rotate.cu` + `InterpolationWrap.hpp`,3 interp + REPLICATE,bit-matching float math;`gold_rotate_res` 暴露 pre-saturate float 供边界分类。

### 2. CUDA → SYCL 核心映射
| CUDA (CV-CUDA) | SYCL (本移植) |
|---|---|
| `__global__` + `<<<grid,block>>>`,`blockIdx.z`=batch | `q.parallel_for(range<1>{N*Hout*Wout}, λ)`,分解 `n/y/x` |
| `CreateInterpolationWrapNHW<T,...>` vector-type 派发(uchar3/float3) | 单 `EltT` 模板 + 运行时 C + 运行时 interp,通道内部循环 |
| `compute_warpAffine`(1-thread device kernel 算 cos/sin) | **host 端 double 算 coeff**,memcpy 到 6-double device buffer |
| `cuda::SaturateCast<T>` | `sat_cast<T>`(`sycl::rint` 半偶舍入 + clamp;F32 恒等) |
| 越界 dest 像素不动(靠调用方缓冲) | 越界写 0(zero-fill rotate-into-canvas) |
| `cudaStream_t stream` | `sycl::queue*`(nullptr 用内部 queue,in-order + `VOX_PROFILE` 开 `enable_profiling`) |
| kernel 返回 void | `parallel_for` 返回 `sycl::event`,保存供 `last_profile()` 计时 |
| 原子 / 共享内存 | **无** —— 纯 per-pixel 寄存器运算 |
| 接口 void / 异常 | 返回 **bool**(校验 C/dtype/shape) |

> 本仓版本为 self-contained header-only:`nv::linear_launch` → 原生 `q.parallel_for`,不依赖项目 `common/launch.hpp`/`common/check.hpp`(后者 `Assertf/Asserts` 未使用,入参校验直接 `return false`)。kernel 数学逐字不变。

### 3. 写出文件
```
OpRotate/
├── README.md                    # 工作说明 + 使用指南
├── 总结.md                      # 本文件(移植总结)
├── build.sh                     # 编译脚本(编 3 个 test 二进制;kernel header-only)
├── rotate.hpp                   # header-only:rotate_kernel(3 interp)+ dispatch + dtype 派发 + 接口/实现
├── rotate_helpers.hpp           # TypeMin/Max/nhwc_ptr/border_replicate/cubic_coeffs(CPU+device safe)
└── test/
    ├── rotate_gold.hpp          # CPU gold(3 interp + REPLICATE;gold_rotate_res 暴露 pre-saturate float)
    ├── test_rotate.cpp          # 合成精确值 vs gold + 负向
    ├── test_rotate_real.cpp     # 随机 23.5° + FP 边界分析 + wall 性能

```
**组织原则**(同 OpResize / OpNMS / OpColorCvt / OpNormalize):kernel 全部 header-only(`rotate.hpp`),test 直接 include,`build.sh` 只编 3 个 test 二进制。原始 NVIDIA 源在 CV-CUDA 0.16.0 仓库,此处不重复收录。

### 4. 编译调试
1. kernel header-only(`rotate.hpp`),test 直接 include,`build.sh` 只编 3 个 test 二进制,无需单独编 kernel。
2. 编译选项 `icpx -fsycl -std=c++17 -O2 -g -I. -ffp-model=precise -fno-fast-math -fimf-precision=high`(由 `build.sh` 编 3 个目标)。
3. 首次板上 build/run 即 clean(RC=0,无 fix)。复用 normalize 的 `make_internal_queue`/`last_profile`/arg-shadow/`sat_cast`/TypeMin/TypeMax/dtype-switch/容差比较模式。

### 5. 正确性验证
40 case(4 dtype × {NEAREST 0/90/45/30/200°, LINEAR 0/180°, CUBIC 0°} × C∈{1,3,4})全部 `=== ALL TESTS PASSED ===`,bit-exact mismatch=0:
- NEAREST 任意角 bit-exact(host 算 coeff → srcX GPU/gold bit-identical → floor 一致)。
- LINEAR@0/180°、CUBIC@0° 权重坍缩为单 1.0 tap → 精确复制。
- 负向:C==2 / C==5 拒绝。

随机(23.5°,N=4 S=128 C=3)全 PASS:
- U8 NEAREST/LINEAR exact;U8 CUBIC maxdiff=1(tol 1, 0 mismatch)。
- F32 NEAREST exact;F32 LINEAR maxdiff=1.19e-7(1 ULP,tol 1e-4);F32 CUBIC maxdiff=2.98e-7(1 ULP,tol 1e-3)。

## 二、性能

### 1. profile 方法
SYCL event profiling(`VOX_PROFILE=1` opt-in,`last_profile(ns)`)+ steady_clock wall。`test_rotate_profile.cpp`:F32,best。

### 2. profile 数据
```
S=128 N=4:  rotate_kernel best=0.0322ms avg=0.0331ms  wall=0.0708ms  overhead=0.0386ms  kernel=2036 Mpix/s  wall=926 Mpix/s
S=720 N=1:  rotate_kernel best=0.2435ms(LINEAR)/0.2434ms(CUBIC)  wall=0.367ms  overhead=0.123ms  kernel=2129 Mpix/s  wall=1412 Mpix/s
```

### 3. 性能极限分析
- **内存带宽受限**:每像素 4(LINEAR)/16(CUBIC)次 REPLICATE 随机读 + 1 写。S=720 时 LINEAR 与 CUBIC kernel 时间几乎相同(CUBIC 16-tap 额外读被 cache/延迟掩盖)→ 带宽非计算受限,tile+RPT 收益可忽略,不做(同 colorcvt/normalize)。
- S=128 时 overhead(coeff memcpy + launch + wait)> kernel;S=720 时 overhead ≈ wall 的 34%。无原子,无中段 host sync(forward 内仅一次 48 字节 coeff memcpy+wait)。
- 后续优化方向(本轮未做):tile + 只读 cache(采样局部性)+ RPT —— 带宽受限下收益有限。

### 4. CPU↔GPU speedup 对比

为回答"算子移植有没有做 CPU/GPU 对比验证",给 `test_rotate_profile.cpp` 加了 env 门控 `VOX_CPUTIME=1`(默认关)的 CPU gold 参考计时。CPU 参考为朴素单线程实现(与正确性测试同一份 `rotate_gold.hpp`,3 interp + REPLICATE),`speedup = cpu_ms / gpu_best_kernel_ms`(SYCL event profiling, best-of-20;CPU best-of-3)。profile 现支持可选第 5 参数 `interp`(0=NEAREST/1=LINEAR/2=CUBIC,默认 LINEAR),GPU forward 与 CPU gold 共用同一运行时 interp。

运行:

```
ONEAPI_DEVICE_SELECTOR=opencl:gpu VOX_PROFILE=1 VOX_CPUTIME=1 ./test/test_rotate_profile <N> <S> <C> <runs> <interp>
```

实测(Intel Arc [0xb080],F32 C3,23.5°,2026-07-02):

| 配置 | gpu best_kernel | cpu_ref (best-of-3) | speedup |
|---|---|---|---|
| S=128 N=4 C=3 LINEAR | 0.0455ms | 0.4867ms | 10.69× |
| S=720 N=1 C=3 LINEAR | 0.2413ms | 4.9858ms | 20.67× |
| S=720 N=1 C=3 CUBIC  | 0.2847ms | 6.5659ms | 23.06× |

说明:
- speedup 反映"GPU vs 朴素单线程 CPU"量级,**非优化基线**(GPU 自身性能/瓶颈见 4.2/4.3)。
- S=128 小图 GPU launch-bound(kernel 0.045ms 中 overhead 占优),speedup 偏低(10.7×);S=720 大图 kernel 主导,speedup 升到 20-23×。
- CUBIC 比 LINEAR speedup 略高:CPU gold 16-tap 比 4-tap 慢得更明显(6.57 vs 4.99ms),而 GPU 16-tap 被 cache 掩盖只略慢(0.285 vs 0.241ms)——GPU 隐藏访存延迟的优势放大。
- CPU 参考非 NVIDIA bit-diff(CV-CUDA rotate 无独立 gold,本 gold 自移植,见三章),逻辑逐字照抄原版。

## 三、验证局限(诚实声明)
1. **自洽验证,非对齐验证**:CV-CUDA rotate 不带独立 test gold,本 gold 是我自己从 `rotate.cu` + `InterpolationWrap.hpp` 移植的 CPU 参考 → 证明「SYCL kernel == 我的 CPU 参考」,**不直接证明 == NVIDIA CV-CUDA 实际输出**。若 kernel 与 gold 都从同一份源理解错,二者会一致地错,测试仍 PASS。缓解:kernel 与原 `.cu` + `InterpolationWrap.hpp` 逐行对照是最强独立检查。
2. **未与 NVIDIA CV-CUDA 做 bit 级 diff**:真正的金标准是同参数喂 NVIDIA `cvcudaRotate` 和本 SYCL kernel 比输出。这一步没做(需 NVIDIA GPU + 编译 CV-CUDA,Intel Arc 跑不了 CUDA 版)。
3. **FP 归因较确定**:LINEAR/CUBIC 仅 mul/add(无 div/sqrt)→ Intel GPU float-**除法** quirk 不适用;残差是 FMA 收缩 ULP 噪声(F32 1 ULP),非除法发散。权重仅乘加,归因较直接观测确定。

## 四、与其它算子移植的差异小结
| 维度 | 体素化 | NMS | CvtColor | Normalize | Rotate |
|---|---|---|---|---|---|
| kernel 数 | 3 | 1 | 11 模板(6 族) | 1 | 1(3 interp 分支) |
| 原子/hash 表 | 有 | 无 | 无 | 无 | 无 |
| 优化阶梯 | opt1-4 | 无 | 无(带宽) | 无(本轮) | 无(带宽) |
| FP 风险 | 点除法→±1 | IoU 除法→翻转 | HSV/YUV 除法 | stddev 1/sqrt ULP | 仅 mul/add,1 ULP FMA 噪声(无除法) |
| 接口返回 | op 拥有输出 | 调用方传 out | bool | bool | bool |
| coeff 计算位置 | — | — | — | per-axis 广播 | **host double**(隔离发散) |

共性:都用 OpenCL 后端;都复用 `make_internal_queue` + `VOX_PROFILE` event 计时;NEAREST/host-coeff 路径可做到 bit-exact;性能极限都属「带宽受限,需算法级改动才能突破」。
